使用教學

為什麼要使用testlink?

Testlink是基於Web的測試案例管理系統

主要功能:測試項目管理、需求管理、測試計劃管理、測試案例的創建、管理和執行，並且還提供了統計功能。

需求文件 + 測試文件的內容都可以在這個系統上來做管理

使用畫面

專案角色

  1. Guest：只有讀的許可權，適合於檢視測試案例和測試需求，以及專案分析的使用者。
  2. Test designer：可以新增測試案例和測試需求。
  3. Tester：只能執行測試案例。
  4. Senior tester：可以檢視和維護測試案例，並且可以執行測試案例，但是不能管理測試計劃、分配測試任務。
  5. Leader：可以新增測試規格和測試需求的所有工作，還可以管理測試計劃、分配測試任務。
  6. Admin：維護產品，使用者。
Leader是各組的組長、Admin權限則是助教、老師才會有，剛創建好的使用者預設是Guest， 所以到時候各組組長要請助教幫你們設成Leader的角色，之後各組組長再去設定小組成員的權限， 角色的部分在看組長要怎麼分配，每個角色有各自的責任

開始建置

1.   設定專案角色

Assign User Roles

2.   建立需求

Requirement Specifications


先建資料夾，在建需求
Document ID就寫SRS文件裡面需求的編號，Title就需求名稱，Status看你有沒有特殊的情況，不然就預設就好， Type也是選Use Case就好，然後最後那邊可以設定這個需求需要用幾個測試案例來測， 在最後產報告的部分，有需求覆蓋率可以看，如果這邊設定5個，然後你只用2個來測的話， 就會是40%。

3.   建立測試計畫

Test Plan Management

Active, Public記得打勾

4.   建立測試案例

Test Specification


先建Test Suit , 再建Test Case
Test Suit就是資料夾
Title, Summary, Preconditions

最下面可以設定重要性、執行方法、預計測試時間
重要性:低中高，執行方法:手動、自動，自動的話必須搭配其他工具來使用，目前用不到。 keyword則是設定關鍵字，方便定義以及查詢這個test case，是testlink方便使用的一個功能， 不過我們這種小規模的其實用不到，旁邊tree直接展開就看完所有test case了，不用再特別查詢。

建立測試步驟以及對應需求
測試步驟可以清楚知道測試案例每一步是怎麼測，還有每個步驟的預期結果

把測試案例加到測試計畫中

5.   設定Build

Builds / Releases


一般Build名稱以日期來建，EX：2019/05/01 Build
表示從什麼時候開始執行測試

6.   分配測試工作

Assign Test Case Execution


分配測試人員

7.   開始測試

Execute Tests

這邊我是用權限tester來做範例，所以功能比較少，介面就只有幾項功能可以使用。

測試結果
可以看到測試的步驟、可能還會有第2步第3步，我這邊因為作範例就沒有建很多。 然後它可以針對每個步驟選擇測試結果(Passed,Failed,Blocked)， 最下面對整個測試案例選擇測試結果

8.   觀看報告

Test Reports

leader可以看到整個專案測試的表格，HTML方式做呈現，可以另外輸出成PDF等格式。 Test Report勾選Test Case related Requirements，這可以輸出測試案例、需求結合的檔案
