使用教學

為什麼要使用testlink?

Testlink是基於Web的測試案例管理系統

主要功能:測試項目管理、需求管理、測試計劃管理、測試案例的創建、管理和執行，並且還提供了統計功能。

需求文件 + 測試文件的內容都可以在這個系統上來做管理

使用畫面

專案角色

  1. Leader：可以新增測試規格和測試需求的所有工作，還可以管理測試計劃、分配測試任務。
  2. Test designer：可以新增測試案例和測試需求。
  3. Senior tester：可以檢視和維護測試案例，並且可以執行測試案例，但是不能管理測試計劃、分配測試任務。
  4. Tester：只能執行測試案例。
  5. Guest：只有讀的許可權，適合於檢視測試案例和測試需求，以及專案分析的使用者。
  6. Admin：維護產品，使用者。
Leader是各組的組長、Admin權限則是助教、老師才會有，剛創建好的使用者預設是Guest， 所以到時候各組組長要請助教幫你們設成Leader的角色，之後各組組長再去設定小組成員的權限， 角色的部分在看組長要怎麼分配，每個角色有各自的責任

開始建置

1.   設定專案角色

Assign User Roles

2.   建立需求

Requirement Specifications


先建資料夾，在建需求

Type選擇Feature或System Function
（主要放功能需求）
Document ID就寫SRS文件裡面需求的編號，Title就需求名稱，Status看你有沒有特殊的情況，不然就預設就好， Type選擇Feature或System Function，有其他別的需求種類再選別的，然後最後那邊可以設定這個需求需要用幾個測試案例來測， 在最後產報告的部分，有需求覆蓋率可以看，如果這邊設定5個，然後你只用2個來測的話， 就會是40%。
